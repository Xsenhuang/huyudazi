 # 《新上海人说上海话》场景化学习产品文档（v0.1）

> 目标：基于《新上海人说上海话》的内容体系，设计一套**可自由选学、偏游戏化**的沪语学习产品结构，让用户“学了能开口、记得住”，而不是“看完就忘”。

---

## 1. 产品目标与核心原则

### 1.1 产品目标

1. 让零基础/有点基础的用户，能在高频生活场景中**用上海话解决实际问题**（打招呼、问路、点餐等）。  
2. 将纸质教材的内容转化为**可交互的“场景技能树”**，并加入**闯关、复习、任务**机制，减少遗忘。  
3. 保持与《新上海人说上海话》在**场景划分与表达方式上的对齐**，方便用户“书＋产品”互相补充。

### 1.2 设计原则

- **场景优先**：围绕“要完成的任务”组织内容，而不是单纯语法点。  
- **小步可用**：每个小场景都提供 2–3 句“万能句”，用户学完即可开口。  
- **重复检索**：通过间隔复习、混合抽查，让已学内容在不同场景下被反复调用。  
- **尊重教材**：所有小场景都标注对应的**书中课次**，实现标准化与可追溯。

## 2. 内容来源与对齐方式

### 2.1 主要内容来源

- 《新上海人说上海话》（上海大学出版社）  
  - 《上海话的发音》  
  - 第1课 至 第20课  
  - 附录：儿歌四则、练习题

### 2.2 对齐策略

1. **产品大类 = 若干课次的合并/重组**  
   - 将教材中主题相近的课次归为同一个「产品大类」，方便用户按生活领域自由选择。  
2. **小场景 = 书中某一课里的具体情境**  
   - 如“大类1：基础发音&打招呼”下的「见面致礼」「自我介绍」等，对应第1–2课的具体片段。  
3. **每个小场景都标注“参考课次”**  
   - 例如：  
     - 小场景「见面致礼」：参考第1课·见面致礼  
     - 小场景「自我介绍&认识」：参考第2课·问人/建立关系

## 4. 通用学习机制：4 关卡 + Boss 战 + 复习

### 4.1 单场景四关卡

每一个「小场景」均按统一的 4 关设计，形成可复用 content 模板：

1. **第1关：听懂（识别）**  
   - 形式：听音频 → 选择中文意思 / 选择下一句 / 判断场景  
   - 目标：用户能在 5–10 秒语料中，快速匹配意思与反应。  
2. **第2关：跟读（模仿）**  
   - 形式：逐句跟读 2–3 句核心句，给“关键词提示”（如：侬好｜好久勿见｜咯）  
   - 目标：先把**节奏＋关键词**练顺，不苛求完全正确发音。  
3. **第3关：替换（半自由输出）**  
   - 形式：给出句型骨架 + 若干可选替换项（人/地/钱/时间等），用户重复 3–5 次。  
   - 目标：让用户真正掌握“句型骨架”，能迁移到别的场景。  
4. **第4关：实战（任务对话）**  
   - 形式：6–10 轮对话，与 NPC 对话树交互，至少 1 个“突发分支”。  
   - 目标：在**接近真实的任务场景**中完成“买到/问到/办成”。

### 4.2 Boss 战（混合复习）

每完成 3–5 个小场景后，触发一次「Boss 战」：

- 混合出题：把不同小场景的句型和任务打乱组合，不按章节顺序。  
- 形式：随机对话、听力抽查、填空等。  
- 作用：利用**混合练习（Interleaving）**增强记忆抗干扰，逼迫用户跨场景调取已有知识。

### 4.3 复习系统（SRS + 卡组）

- 每个小场景结束后，生成一个**“复习卡组”**：  
  - 听懂卡：听音频 → 选意思  
  - 产出卡：给中文意图 → 说/选沪语句子  
- 间隔复习：系统按时间间隔（例如 1天、3天、7天）推送对应卡组。  
- 形式上做成「30–60 秒速通练习」，完成后奖励经验值/连胜，避免“刷题感”。

## 5. 统一掌握标准（Mastery）

所有小场景采用**统一的“掌握判定”**，避免只是“学过”：

1. **5 秒反应能力**  
   - 看到中文意图（如“打招呼并问近况”），用户在 5 秒内能说出至少 1 句场景“万能句”。  
2. **替换迁移能力**  
   - 针对某个句型骨架，连续做 3 次替换（人物/地点/时间等），正确 ≥ 2 次。  
3. **随机抽查通过率**  
   - 在 Boss 战或日常复习中，场景被“打乱混合”抽到时，能正确接话 ≥ 2 次。  
4. （可选）**24 小时回访**  
   - 隔天 1 分钟快速复习，正确率 ≥ 80% 依然视为“掌握稳定”。

可以在前端以 3 级标签展现：

- **L1 听得懂、跟得上**（听懂 + 跟读通过）  
- **L2 说得出、用得上**（替换关通过）  
- **L3 能应变、能实战**（实战关 + 随机抽查通过）

每个小场景统一包含：

6. **示例对话（6–10 轮）**：对应第4关，任务闭环。  
7. **四关卡题型示例**：听懂、跟读、替换、实战。  
8. **掌握检测标准**：该场景的 Mastery 判定。

## 6.4 场景打样 A：见面致礼（第1课）

### 6.4.1 场景信息

- 场景名：**见面致礼**  
- 参考课次：第1课·见面致礼  
- 典型场景：电梯相遇、楼下遇到邻居、公司门口遇到同事。  
- 场景目标：  
  - 完成：**打招呼 → 询问近况 → 客气回应 → 自然结束对话**。

### 6.4.2 三句万能句

1. 侬好！好久勿见咯。  
2. 侬近来哪能？  
3. 蛮好蛮好，下趟再聚。

### 6.4.3 高频词/短语（示例 10 个）

- 侬好 / 阿拉 / 伊  
- 好久勿见  
- 近来  
- 忙伐  
- 还好 / 蛮好  
- 辛苦侬咯  
- 谢谢侬  
- 下趟  
- 再会  

### 6.4.4 礼貌点

- **“辛苦侬咯”**：对对方付出表示体恤，比单纯“谢谢侬”更有人情味。

### 6.4.5 易混点

- 问近况不宜太“查户口”，场景中默认只问：  
  - 轻量版：侬近来哪能？忙伐？  
  - 避免一上来问收入、家庭等高隐私内容。

### 6.4.6 示例对话（8 轮）

1. A：侬好！好久勿见咯。  
2. B：侬好侬好！真个好久勿见。  
3. A：侬近来哪能？忙伐？  
4. B：还好还好，近来有点忙。  
5. A：辛苦侬咯。  
6. B：勿辛苦，习惯咯。侬呢？  
7. A：阿拉也还好。  
8. B：蛮好蛮好，下趟再聚。再会！

### 6.4.7 四关卡题型示例

**第1关：听懂（识别）**  
- 听到“侬近来哪能？”→ 选中文意思：  
  - 正确答案：你最近怎么样？  
- 听到“下趟再聚”→ 选最合适回应：  
  - 正确答案：好个 / 好的。  
- 听到“辛苦侬咯”→ 判断语气：  
  - 正确答案：表达体恤。

**第2关：跟读（模仿）**  
- 跟读句 1：侬好｜好久勿见｜咯  
- 跟读句 2：侬近来｜哪能  
- 跟读句 3：蛮好｜下趟再聚  
- 判定：关键词命中率 ≥ 80% 视为通过。

**第3关：替换（半自由输出）**  
- 骨架：侬近来【哪能】？【忙伐】？  
- 提供若干替换词（哪能 / 还好 / 蛮好；忙伐 / 最近忙伐等），做 5 次。  
- 通过：5 次中 ≥ 4 次表达完整自然。

**第4关：实战（对话推进）**  
- 任务：在电梯遇到熟人，完成“打招呼→问近况→结束”。  
- 要求：用户至少主动说出 2 句“万能句”。  
- 分支：对方说“很忙要走了”，用户应使用“理解 + 结束句”，如：  
  - 好个好个，侬先忙，下趟再讲。  
- 判定：完成任务 + 主动句 ≥ 2。

### 6.4.8 掌握标准（该场景）

1. 在看到“电梯遇到熟人”这一提示后，5 秒内能说出“打招呼+问近况”两句。  
2. 替换句型连续 3 次，正确表达 ≥ 2 次。  
3. 在 Boss 战混合抽查中，被随机问到“结束对话”意图时，能给出合适结束句 ≥ 2 次。

## 6.5 场景打样 B：自我介绍 & 认识（第2课）

### 6.5.1 场景信息

- 场景名：**自我介绍 & 认识**  
- 参考课次：第2课·问人 / 建立关系  
- 典型场景：朋友介绍新朋友、邻居初次见面、新同事入职认识。  
- 场景目标：  
  - 完成：**自我介绍 → 问对方基本信息 → 找共同点 → 自然约定后续联系**。

### 6.5.2 三句万能句

1. 阿拉姓X，叫X。  
2. 侬做啥个工作个？  
3. 蛮高兴认得侬，后头有空一道出来白相。

### 6.5.3 高频词/短语（示例 10 个）

- 姓… / 叫…  
- 哪能称呼  
- 做啥个（工作）  
- 上班勒啥地方  
- 住勒哪能  
- 老早（一早/以前）  
- 一道（一起）  
- 白相（玩）  
- 加个微信  
- 方便伐

### 6.5.4 礼貌点

- **先问“哪能称呼”再问具体信息**  
  - 如：我哪能称呼侬？ → 再问做啥工作 / 上班勒啥地方，显得更礼貌。

### 6.5.5 易混点

- 问住址、收入等信息容易越界：  
  - 产品内默认只引导问**称呼 + 工作/身份**，把“住哪里”作为进阶/可选内容。

### 6.5.6 示例对话（9 轮）

1. A：侬好，我阿拉姓张，叫张XX。  
2. B：侬好侬好，我姓李，叫李XX。  
3. A：我哪能称呼侬？叫侬小李可以伐？  
4. B：可以个。  
5. A：侬做啥个工作个？  
6. B：阿拉做设计个。侬呢？  
7. A：阿拉做产品个。蛮高兴认得侬。  
8. A：方便伐？加个微信，后头一道出来白相。  
9. B：好个好个。（分支：如果 B 说“不方便”，A 应用委婉结束句）

### 6.5.7 四关卡题型示例

**第1关：听懂（识别）**  
- 听到“阿拉姓张，叫张XX”→ 选意思：  
  - 正确：我姓张，叫张XX。  
- 听到“侬做啥个工作个？”→ 判断是在问：  
  - 正确：工作。  
- 听到“方便伐？加个微信”→ 选最合适回应：  
  - 正确：好个。

**第2关：跟读（模仿）**  
- 跟读 1：阿拉｜姓X｜叫X  
- 跟读 2：侬做啥个｜工作个  
- 跟读 3：方便伐｜加个微信  
- 判定：关键词命中率 ≥ 80%。

**第3关：替换（半自由输出）**  
- 骨架 1：阿拉姓【姓】，叫【名】。  
- 骨架 2：侬做【行业/岗位】个？  
- 提供姓氏（张/王/李/陈）与岗位（设计/老师/工程师/销售/学生）词包。  
- 通过：5 次替换中 ≥ 4 次完整自然。

**第4关：实战（对话推进）**  
- 任务：在朋友介绍下认识新朋友，完成“自我介绍→问工作→约后续联系”。  
- 要求：至少主动说出 3 句关键句（包括万能句1、2、3中的任意三句）。  
- 分支：对方说“我赶时间”或“不方便加微信”，用户需用委婉结束句：  
  - 例如：“好个，侬先忙，下趟再讲。”  
- 判定：完成任务 + 主动句 ≥ 3。

### 6.5.8 掌握标准（该场景）

1. 在“朋友介绍认识”的提示下，5 秒内能完成一整句自我介绍。  
2. 能基于骨架句“侬做啥个工作个？”做 3 次岗位替换，≥ 2 次表达正确。  
3. 在混合复习中，遇到“加微信被拒绝/对方赶时间”分支时，能用委婉结束句接住 ≥ 2 次。

   - v0.2：补齐更多大类的详细打样；  
   - v0.3：加入完整复习策略和运营活动（打卡任务、线下挑战等）。
